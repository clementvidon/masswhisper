# Cross-Workspace Documentation

## Documentation structure

- `docs/architecture/` → architecture, system boundaries, and cross-workspace design decisions
- `docs/ops/` → deployment model and ops architecture
- `docs/security/` → public exposure and security constraints
- `docs/frontend/` → frontend local workflows and shared routing design
- `docs/manifest/` → manifest and topic config formats used by deployment tooling
- `docs/convention/` → repo conventions for naming, formats, and documentation
- `docs/workflow/` → development workflows
- `docs/runbooks/` → operational procedures for the dedicated deployment model
